0 Bricks4Agent 是什麼

一句話：broker-centered governed AI operations platform（broker 中介的、受治理的 AI operation 平台）。

三層架構

1. 高階模型層（high-level）：負責對話、釐清、查詢改寫、意圖判斷。可以「解讀」、不可以自己執行 side effect。

2. broker / control plane：核心、負責 workflow gating、ACL/capability 授權、稽核鏈、artifact 派發、管理介面。不該像自主 planner、它的價值是約束行為、讓行為可審計可重現。

3. execution layer：被治理的 agent runtime、worker process、tool 路由、deploy 執行、browser 執行。消費「結構化意圖」、不是原始對話。

主入口流程（canonical live path）

LINE webhook
  → line-worker
  → broker /api/v1/high-level/line/process
  → HighLevelCoordinator

所有 LINE 對話走這條。其他 entry（例如 dashboard 直接戳）也經 broker、但 LINE 是 design 的主場景。

詳細架構讀 docs/reports/CurrentArchitectureAndProgress-2026-03-26.md（組長最近一次完整 architecture writeup）。

1 Docker 心智模型

類比

Docker container ≈ 一台獨立的迷你電腦。在你 Windows 跑 N 個迷你 Linux、每個跑不同 service、互不影響、關掉不影響桌面。

Image = 「裝好系統的光碟」；container = 「塞光碟開機後的那台機器」。Image 是模板、container 是 instance。

為什麼 broker 用 Docker

  • broker（C# / .NET 8）+ worker（多個獨立 process）+ SQLite + cache — 多 service、本機裝會打架
  • 給新組員 docker-compose.yml、敲一行就跑起來、跟你環境一致
  • 部署 VPS 跟本機同一份 image、reproducible

3 個需要懂的詞

名詞白話
image「光碟」、不可變的軟體包
container「開機後的迷你電腦」、image 的 runtime instance
docker compose「樂高說明書」、多 container 怎麼相互連線、env / volume / port

4 設定環境變數 10min

broker 啟動需要密鑰跟設定。在 tools/ 建一份 .env.trading：

複製範例（如果 repo 有 .env.trading.example）、或直接建新檔

最少要設這 3 個

# 1. Broker master key (32 byte base64) — at-rest 加密用
#    PowerShell 產一組：
#    [Convert]::ToBase64String((1..32 | %{Get-Random -Max 256}))
BROKER_MASTER_KEY_BASE64=（貼上面產的）

# 2. Scoped token secret (隨機長字串、HMAC 用)
BROKER_SCOPED_TOKEN_SECRET=（貼另一組）

# 3. LLM API key (gpt-5.4-mini / Gemini / 任何 openai-compatible)
LLM_API_KEY=（你的 key）
.env.trading 絕對不要 commit 進 git。已在 .gitignore 排除、push 前確認。

其他選填

  • HIGH_LEVEL_LLM_API_KEY — HighLevelCoordinator 用、gpt-5.4-mini 預設
  • LINE channel access token / secret — 跑 line-worker 才需要

5 啟動 broker 5min

進 tools/ 跑：

docker compose --env-file .env.trading -f compose.trading.yml up -d broker

各段：

  • docker compose — 用 compose 工具
  • --env-file .env.trading — 載入剛改的環境變數（絕對不能漏）
  • -f compose.trading.yml — 哪個 compose 檔
  • up -d — 起動、-d 是背景跑
  • broker — 只起 broker 一個 service
docker ps 看到一行 broker、status Up X seconds
第一次跑會慢（要 build image + 下載 NuGet + 編譯 C#、10-15 分鐘）。後面再跑都秒開。

要起完整 stack（line-worker + 其他 service）

# 直接 up、不指定 service 名 = 起所有 service
docker compose --env-file .env.trading -f compose.trading.yml up -d

7 常用 Docker 指令速查

場景指令
看哪些 container 在跑docker ps
看全部（含停的）docker ps -a
看 broker log（最近 50）docker logs broker --tail 50
持續看（Ctrl+C 離）docker logs broker -f
進 container 內 bashdocker exec -it broker bash
停 containerdocker stop broker
重啟docker restart broker
停全部 stackdocker compose -f compose.trading.yml down
code 改完重 builddocker compose --env-file .env.trading -f compose.trading.yml build broker
build 完套用docker compose --env-file .env.trading -f compose.trading.yml up -d --no-deps broker
「--env-file .env.trading 不能漏」 — 漏了 broker 啟動會 crash（master key blank）。每次 compose 指令都加。

8 卡住怎麼辦

狀況 A：docker ps 顯示 Restarting (139)

  1. docker logs broker --tail 50 看錯訊息
  2. 八成是 Broker master key must be configured → 漏了 --env-file
  3. 停掉重起、記得帶 --env-file .env.trading

狀況 B：build 失敗、抓不到 NuGet

  1. 檢查網路 / 關 VPN
  2. 清 cache 重來：docker compose ... build --no-cache broker

狀況 C：登入帳號密碼錯

  1. 預設 prn_dashboard / admin、第一次登入強制改
  2. 忘記？刪 volume 重來（會清資料）： 
    docker compose -f compose.trading.yml down
docker volume rm $(docker volume ls -q | grep broker-data)
# 重新 up

狀況 D：dashboard KPI 全顯示「—」

broker 沒接到資料源（worker 沒起 / health endpoint fail）。單獨跑 broker 不會有 worker、屬正常。要全 stack 跑 docker compose ... up -d（不指定 service）。

狀況 E：完全動不了、想砍掉重來

docker compose -f compose.trading.yml down
docker volume rm $(docker volume ls -q | grep broker-data)
docker rmi $(docker images -q | head -5)  # 砍 image (可選)
docker compose --env-file .env.trading -f compose.trading.yml up -d broker

還是不行？

  • 截圖 docker ps + docker logs broker --tail 50 丟群組
  • 講清楚卡在第幾章節
  • 絕對不要貼 .env.trading 內容（密鑰會外洩）
