0 心智模型：Docker 是什麼

類比一

Docker container ≈ 一個獨立的迷你電腦。你可以在你的 Windows 上跑 N 個迷你 Linux、每個跑不同程式、互不影響。關掉迷你電腦不會影響你 Windows 桌面。

類比二

Image ≈ 「裝好系統 + 軟體的光碟」、Container = 「把光碟塞進去開機後的那台機器」。Image 是模板、container 是 instance。

為什麼這個專案用 Docker？

  • broker（C# / .NET 8）+ worker（Python / Node）+ SQLite + cache — 一堆服務、本機裝會搞死
  • 給其他組員一份 docker-compose.yml、他們敲一行 docker compose up 就跟你跑一樣的環境
  • 部署到 VPS 也是同一個 image、本機跑得起來＝VPS 跑得起來

你需要懂的 3 個名詞

名詞白話
image「光碟」、不可變的軟體包
container「開機後的迷你電腦」、用 image 起動的執行 instance
docker compose「樂高說明書」、定義 N 個 container 怎麼相互連線、變數怎麼塞、volume 怎麼掛

1 裝 Docker Desktop 10min

下載 Docker Desktop for Windows
安裝、重開機（一定要重開、不然 WSL2 沒生效）
打開 Docker Desktop、左下角看到綠色「Engine running」
PowerShell / cmd 跑 docker --version 應該回 Docker version 27.x.x ...
Windows 家用版要先啟用 WSL 2 + Hyper-V。Docker Desktop 安裝時會自動引導、跟著做就好。如果跳「WSL 2 installation is incomplete」、跟提示連結裝個 wsl --update 後重開。

2 拿 source code 5min

先裝 Git（如果還沒有）→ git-scm.com
PowerShell 開到你想放專案的資料夾（例 Desktop）： 
cd C:\Users\你\Desktop
git clone https://github.com/forself/Bricks4Agent.git
cd Bricks4Agent
應該看到一堆檔案、特別是 packages/csharp/broker/、tools/compose.trading.yml
如果 GitHub 要登入、可以用個人帳號（fork 過更好、push 不會打架）。

3 設定環境變數 .env.trading 10min

compose 啟動需要一些密鑰跟設定。先複製範例檔：

PowerShell： 
cd tools
copy .env.trading.example .env.trading
（如果 example 不存在、跟組長拿一份）

最少要設這 3 個（用 notepad 開 .env.trading）

# 1. Broker master key — 32 byte 隨機字串、base64 編碼
#    PowerShell 產一組：
#    [Convert]::ToBase64String((1..32 | %{Get-Random -Max 256}))
BROKER_MASTER_KEY_BASE64=（貼上面產的）

# 2. Scoped token secret — 同樣方法產
BROKER_SCOPED_TOKEN_SECRET=（貼上面產的）

# 3. LLM API key — 給 broker 內部 LLM 呼叫用、可以先用 Gemini free tier
#    https://aistudio.google.com/apikey 拿
LLM_API_KEY=（貼 Gemini key）
這個檔案絕對不要 commit 進 git。.gitignore 已經排除、但確認一下。
其他像 BingX / Alpaca API key 是進階用、暫時可不填、bot 那邊會 disable。

4 一鍵啟動 broker stack 5min

在 tools/ 資料夾、PowerShell 跑：

docker compose --env-file .env.trading -f compose.trading.yml up -d broker

解析每段：

  • docker compose — 用 compose 工具
  • --env-file .env.trading — 載入你剛改的環境變數（絕對不能漏）
  • -f compose.trading.yml — 用哪個說明書
  • up -d — 起動、-d 是 detach（背景跑）
  • broker — 只起 broker 這一個 service、不起其他附屬
docker ps 應該看到一行 b4a-broker、status Up X seconds
第一次跑會慢（要 build image、10-15 分鐘下載 + 編譯）。後面再跑都是秒開。

5 開瀏覽器確認 dashboard 2min

瀏覽器網址列輸入： 
http://localhost:5100
看到 「B4A Broker」 登入頁面、預設帳號 prn_dashboard / 密碼 admin
登入後、第一次會強制改密碼、改一個你記得的
切到 📊 儀錶板 tab、應該看到大型 health gauge + 14 個 agent 燈陣
如果瀏覽器跳「無法連線」、container 還沒起來、等 1 分鐘再試。docker logs b4a-broker --tail 30 看 log 確認。

6 常用 Docker 指令速查

場景指令
看哪些 container 在跑 docker ps
看所有 container（包含停掉的） docker ps -a
看 container log（最近 50 行） docker logs b4a-broker --tail 50
持續看 log（Ctrl+C 離開） docker logs b4a-broker -f
進 container 內 bash 看裡面（debug 用） docker exec -it b4a-broker bash
停掉某個 container docker stop b4a-broker
重啟某個 container docker restart b4a-broker
停掉所有 stack 的 container docker compose -f compose.trading.yml down
code 改了、要重 build broker image docker compose --env-file .env.trading -f compose.trading.yml build broker
build 完套用新 image docker compose --env-file .env.trading -f compose.trading.yml up -d --no-deps broker
記住：「--env-file .env.trading 不能漏」。漏了 broker 啟動會 crash（master key blank、抓不到 secret）。每次跑 compose 指令都加。

7 我卡住了怎麼辦

狀況 A：docker ps 顯示 Restarting (139)

  1. docker logs b4a-broker --tail 50 看錯訊息
  2. 八成是 Broker master key must be configured → 漏了 --env-file
  3. 停掉 + 重起時記得帶 --env-file .env.trading

狀況 B：build 失敗、抓不到 NuGet package

  1. 檢查網路：ping nuget.org
  2. 如果用 VPN、暫時關掉
  3. 清 build cache 重來：docker compose ... build --no-cache broker

狀況 C：登入頁出來但帳號密碼錯

  1. 預設是 prn_dashboard / admin
  2. 第一次登入會強制改密碼、改完才能用
  3. 如果忘記密碼、刪 volume 重來： 
    docker compose -f compose.trading.yml down
docker volume rm b4a-trading_broker-data
# 然後重新 up
    （注意：會清掉所有資料、包含 audit chain）

狀況 D：dashboard 上 KPI 全顯示「—」

  1. broker 沒接到資料源（worker 沒起來 / health endpoint fail）— 正常、單獨跑 broker 不會有 worker
  2. 不影響你看 dashboard / admin panel UI
  3. 之後要全 stack 跑：docker compose ... up -d（不指定 service 名稱）

狀況 E：完全動不了、想砍掉重來

# 停掉所有相關 container
docker compose -f compose.trading.yml down

# 砍掉 volume（會清資料）
docker volume rm b4a-trading_broker-data

# 砍掉 image
docker rmi b4a-broker:latest

# 重新 build + 起
docker compose --env-file .env.trading -f compose.trading.yml up -d broker

還是不行？

  • 把 docker ps + docker logs b4a-broker --tail 50 的截圖丟群組
  • 講清楚你卡在哪一個 step（文件編號）
  • 絕對不要貼 .env.trading 內容（密鑰會外洩）
