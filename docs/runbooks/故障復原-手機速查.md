# 🚑 B4A 故障 → 復原 手機速查

> 出事時先看這頁。手機可讀。詳細逐步 → `Backup-Restore-Runbook.md`。
> 更新 2026-05-30。

---

## 🧯 先安心:真錢有底線

- 真錢持倉有**交易所原生硬止損**(掛在 BingX 上)
- broker 掛掉 / 重啟期間,止損**照樣由交易所擋**
- 所以「系統故障」**≠「真錢裸奔」**
- 最壞情況:trailing 暫停上移(少賺、不會爆倉)

---

## ⏱️ 30 秒分流(出事先對號)

- **broker 還在、只是慢或錯** → 多半自己會好(看「自動修復」)
- **收到 🔴 CRITICAL 推播** → 看「警報對照」是哪種
- **完全沒聲音 / dashboard 連不上** → 可能整台掛 → **場景 C**
- **磁碟滿的警報** → **場景 B**(清空間,不是還原)
- **broker 一直重啟救不回** → **場景 A**(還原 DB)

---

## 🤖 系統自動做的(你不用動)

由內到外分層:

1. 容器崩潰 → docker 自動拉起
2. broker 不健康 → autoheal 約 15 秒重啟
3. worker 卡死 → host watchdog 偵測並重啟
4. bot-node 連線斷 → **自動重連**(broker 重啟後自己接回)
5. 每天 18:00(UTC)→ 自動備份(本機 + R2 離機)

## 📢 警報對照(你會收到什麼)

**🟡 黃色 = 知道就好**
- 價格告警
- 某 worker 重啟了一次

**🔴 CRITICAL = 會 @here 吵醒你**
- broker 掛掉
- crash-loop(重啟救不回、要人介入)
- 心跳停止(AutoTrader 沒在跑)
- circuit breaker 觸發(虧損/DD 惡化)

走 Discord 推播。
(可選加強:healthchecks.io 外部 dead-man,連「整台 VPS 掛、Discord 也發不出」都能告警 → 尚待設定)

## 🆘 四種復原場景

### A — broker 一直 crash(VPS 還活)
DB 壞了。用**本機**備份還原:
- 停 broker
- 從 `/opt/b4a-backups` 取最新、驗 integrity
- 換掉 volume 裡的 broker.db(先刪舊 -wal/-shm)
- 起 broker
- 👉 不碰 R2、不用 go-bag

### B — 磁碟滿(還原沒用)
清空間:
```
docker builder prune -af
docker image prune -af
```
根治:設 daemon.json build cache 上限。

### C — 整台 VPS 沒了
新機一鍵重建:
```
bash bootstrap-restore.sh
```
- 跟著提示貼 **go-bag 的 R2 鑰匙 + git URL**
- 它自動:設 rclone → 抓 R2 備份 → 放回 DB/設定 → **還原 tunnel(DNS 免改)** → 起服務
- 起來後**人工核對持倉**,才手動武裝真錢

### D — 退回某個時間點
```
bash restore-fetch.sh
```
(預設抓最新;要舊的就先列出挑日期)

## 🎒 go-bag(存手機 / 密碼管理器,別只存 VPS)

VPS 死了你會需要、而它們不能跟著死機一起沒:

- ⭐ **R2 鑰匙**:Access Key / Secret / endpoint(沒這個抓不到備份 = 最致命)
- **git URL**(myorigin)
- **Contabo 登入**(開新機)
- **BingX 登入**(核對持倉)
- **Cloudflare 登入**(備援)
- ✅ tunnel 憑證已隨備份,不用另存

## ⌨️ 常用指令(複製即用)

看 broker 健康:
```
curl -s localhost:5100/api/v1/health
```
手動備份一次:
```
bash /opt/b4a/scripts/backup-daily.sh
```
抓最新備份 + 驗:
```
bash /opt/b4a/scripts/restore-fetch.sh
```
新機災難重建:
```
bash bootstrap-restore.sh
```

## 🔑 三個心智模型

1. **重啟 ≠ 還原**
   資料在 docker volume、重開機不會掉。重啟後 broker 自己續跑,通常不用抓備份。

2. **真相在交易所**
   broker 是「控制器」。還原救回的是治理/設定狀態,**不是持倉**;持倉永遠以 BingX 為準,broker 開機自動對帳。

3. **沒演練的災備 = 沒有災備**
   `bootstrap-restore.sh` 還沒在真新機跑過。找台拋棄式 VPS 演練一次,才算真的有 DR。
