# 安全審查清單 (Security Review Checklist)

## 概述

本清單用於審查 Bricks4Agent 各語言實作的安全性，確保所有模組符合安全最佳實踐。

---

## 1. 認證與授權 (Authentication & Authorization)

### 1.1 密碼處理
- [ ] 使用安全的雜湊演算法 (PBKDF2/bcrypt/Argon2)
- [ ] 足夠的迭代次數 (≥100,000 for PBKDF2)
- [ ] 使用隨機鹽值 (≥16 bytes)
- [ ] 密碼比對使用常數時間函數
- [ ] 禁止明文儲存密碼
- [ ] 密碼強度驗證

### 1.2 MFA 實作
- [ ] TOTP 符合 RFC 6238
- [ ] 使用安全的隨機數生成器產生密鑰
- [ ] 密鑰儲存已加密
- [ ] 復原碼使用一次性設計
- [ ] 復原碼已雜湊儲存
- [ ] 限制 MFA 嘗試次數

### 1.3 Session 管理
- [ ] Session ID 使用安全隨機數
- [ ] Session 有適當的過期時間
- [ ] 支援 Session 撤銷
- [ ] 登出時正確銷毀 Session
- [ ] 敏感操作需重新驗證

### 1.4 JWT 實作
- [ ] 使用強加密演算法 (RS256/ES256)
- [ ] 適當的 Token 有效期
- [ ] 包含必要的 Claims
- [ ] 驗證所有 Claims
- [ ] Refresh Token 安全儲存

---

## 2. 輸入驗證 (Input Validation)

### 2.1 一般輸入
- [ ] 驗證所有用戶輸入
- [ ] 使用白名單驗證
- [ ] 驗證資料類型和範圍
- [ ] 限制輸入長度
- [ ] 處理特殊字元

### 2.2 SQL Injection 防護
- [ ] 使用參數化查詢
- [ ] 禁止字串拼接 SQL
- [ ] ORM 正確使用
- [ ] 動態查詢有額外驗證

### 2.3 XSS 防護
- [ ] 輸出編碼 (HTML/JS/URL)
- [ ] Content-Type 正確設定
- [ ] CSP Header 設定
- [ ] 避免使用 innerHTML

### 2.4 正則表達式
- [ ] 設定超時限制 (ReDoS 防護)
- [ ] 避免複雜的巢狀量詞
- [ ] 輸入長度限制

---

## 3. 加密 (Cryptography)

### 3.1 加密演算法
- [ ] 使用現代加密標準 (AES-256, RSA-2048+)
- [ ] 禁止使用過時演算法 (MD5, SHA1, DES)
- [ ] 使用安全的操作模式 (GCM, CTR)
- [ ] IV/Nonce 不重複使用

### 3.2 金鑰管理
- [ ] 金鑰安全儲存
- [ ] 金鑰不寫入日誌
- [ ] 支援金鑰輪換
- [ ] 環境變數/密鑰管理服務

### 3.3 隨機數生成
- [ ] 使用密碼學安全的 PRNG
- [ ] 禁止使用 Math.random() 等
- [ ] 足夠的隨機性

---

## 4. 錯誤處理 (Error Handling)

### 4.1 錯誤訊息
- [ ] 不洩漏敏感資訊
- [ ] 不洩漏堆疊追蹤
- [ ] 不洩漏系統路徑
- [ ] 統一的錯誤格式

### 4.2 日誌安全
- [ ] 不記錄密碼
- [ ] 不記錄完整 Token
- [ ] IP 地址適當遮蔽
- [ ] Email 適當遮蔽
- [ ] 敏感資料脫敏

---

### 5.1 限流實作
- [ ] 滑動視窗演算法正確
- [ ] 計數器執行緒安全
- [ ] 適當的限流閾值
- [ ] 記憶體使用有限制

### 5.2 帳戶保護
- [ ] 登入失敗鎖定
- [ ] 漸進式鎖定
- [ ] 可疑活動偵測

- [ ] 請求大小限制
- [ ] 連線數限制
- [ ] 超時設定

- [ ] 識別所有敏感資料
- [ ] 傳輸加密 (TLS)
- [ ] 儲存加密
- [ ] 存取控制

### 6.2 GDPR/隱私
- [ ] 支援資料刪除
- [ ] 支援資料匯出
- [ ] 資料最小化原則
- [ ] 保留期限設定

- [ ] 檢查已知漏洞 (CVE)

- [ ] 鎖定版本號
- [ ] 定期更新

### 7.2 供應鏈安全
- [ ] 驗證套件來源
- [ ] 檢查套件完整性
- [ ] 最小化相依

## 8. API 安全 (API Security)

### 8.1 認證
- [ ] 所有端點需認證
- [ ] 適當的權限檢查
- [ ] CORS 正確設定
- [ ] CSRF 保護

### 8.2 資料驗證
- [ ] 請求體驗證
- [ ] 回應不洩漏敏感資料
- [ ] 適當的 HTTP 狀態碼

### 8.3 文件安全
- [ ] API 文件不包含敏感資訊

## 9. 程式碼品質 (Code Quality)

### 9.1 安全編碼
- [ ] 無硬編碼的密鑰/密碼
- [ ] 無不安全的反序列化
- [ ] 無命令注入風險
- [ ] 無路徑遍歷風險

### 9.2 執行緒安全
- [ ] 共享資源正確鎖定
- [ ] 避免競態條件
- [ ] 適當的並發控制

## 10. 各語言特定檢查

### 10.1 Rust
- [ ] 適當使用 unsafe
- [ ] 記憶體安全
- [ ] 錯誤處理完整
- [ ] 無 panic 在生產代碼

### 10.2 Python
- [ ] 無 eval/exec 使用
- [ ] pickle 安全使用
- [ ] 適當的異常處理
- [ ] 類型提示完整

### 10.3 Node.js/TypeScript
- [ ] 無 eval 使用
- [ ] 無 Function() 建構
- [ ] 原型污染防護
- [ ] 嚴格模式啟用

### 10.4 前端 JavaScript
- [ ] 無 innerHTML 直接賦值
- [ ] 無 document.write
- [ ] localStorage 敏感資料加密
- [ ] postMessage 來源驗證

## 問題嚴重度定義

| 等級 | 定義 | 修復時限 |
|------|------|---------|
| 🔴 Critical | 可被遠端利用，直接危害系統或資料 | 立即 |
| 🟠 High | 嚴重漏洞，需要特定條件利用 | 24小時 |
| 🟡 Medium | 中等風險，可能導致資訊洩漏 | 7天 |
| 🟢 Low | 低風險，最佳實踐建議 | 30天 |
| ⚪ Info | 資訊性建議，非安全問題 | 可選 |
